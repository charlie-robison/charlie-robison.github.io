Over the beginning of the summer I collaborated with an artist who is a friend of mine on an iOS RPG game. We were both passionate about making a video game and having the ability to play it on iPhone. I had previous experience with XCode and the swift programming language but this would be my biggest project so far using this language. The features of the game included, player movement, picking up items and storing it in a inventory which could be accessed, enemy AI, player attacking capabilities, etc. My biggest take away from this project was how to take on big project such as this. Throughout the process I learned how to organize all my files and create seperate objects, methods and helper methods in my code for each part of the game. For example, the player had its own object which contained its level statistic, its image, and getter/setter methods to create animations. Another example is having a seperate function to create the player's health meter and a seprate function for the enemy movement and AI. Since this game is still in dveleopment, I plan on adding multiplyaer functionality once I learn more about real time APIs. In all, I learned the most about using object oriented programming in a real setting and applied it to making a large video game.
