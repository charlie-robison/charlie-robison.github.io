## UI Creation Made Easy

I always thought, how can I develop a nice-looking, attractive webpage using HTML and CSS? I was confused at first about how developers made beautiful web pages with just the use of these two tedious languages in an efficient manner. When I was introduced to Semantic UI in my Software Engineering class, I finally started to understand how a webpage can be developed nicely in an efficient manner. Semantic UI is a UI framework that uses a library of CSS classes to help design nice-looking UI efficiently within HTML tags. Despite its sharp learning curve, Semantic UI was an eye-opener to me and I view it as a game-changer to developing nice-looking web pages.

## Learning Semantic UI

When I first was learning Semantic UI, I felt overwhelmed with a lot of the content it provided such as all the different CSS classes and elements it had. With that said, once I understood the pattern in which the classes are indicated to describe the type of UI needed to generate, I gained some ground on Semantic UI and started having fun with it. What I enjoy about Semantic UI is its simplicity. CSS classes are written in plain English which are in the form of describing what the UI should look like. This is much easier than using plain HTML and CSS because rather than trying to figure out the specifics of what something should look like using many HTML tags and CSS selectors, Semantic UI can do it in one class. With that said, Semantic UI to me was a big game-changer in frontend web development.

## Major Improvements with Semantic UI

Yes, Semantic UI is frustrating to learn at first, the benefits of learning it outweighs the cost of frustration. Using plain HTML and CSS is not only long and tedious to develop web pages but it's also very hard to get UI on a webpage to look nice. Semantic UI automatically makes UI nice just with the default classes. Of course, you might want to adjust some of the defaults to make the UI more customized and nicer, but the point is, you can focus more on customizing nice-looking UI rather than pounding your head on a desk trying to make the UI decent. ### HTML and CSS Code for Menu Bar in Plain HTML/CSS Webpage
#### HTML
```
  <ul id="nav">
    <li><a href="#intro">Introduction</a></li>
    <li><a href="#ie">Internet Explorer</a></li>
    <li><a href="#fox">Firefox</a></li>
    <li><a href="#chrome">Chrome</a></li>
  </ul>
```

#### CSS
```
    #nav ul {
      list-style-type: none;
      padding: 0;
      margin: 0;
    }
    
    #nav li {
      display: inline;
      margin: 75px;
    }
```


### HTML and CSS Code for Menu Bar in Semantic UI Webpage
#### HTML
```
    <div class="ui fixed top inverted four item menu">
      <a href="#intro" class="item">Introduction</a>
      <a href="#ie" class="item">Internet Explorer</a>
      <a href="#fox" class="item">Firefox</a>
      <a href="#chrome" class="item">Chrome</a>
    </div>
```

#### CSS
```
    /* No CSS USED! */
``` 

As you can see, UI design using Semantic UI is much more efficient and much nicer than plain HTML and CSS. Once I discovered how much easier and nicer this way, I thought of Semantic UI as a game-changer despite its touch learning curve.

## The Impact of UI Frameworks

With all this kept in mind, there are many benefits of using a UI framework such as Semantic UI in software engineering. as stated above, UI frameworks provide make UI development more efficient, easier, and the final product is much nicer. I thought it would take years to develop a nice-looking webpage using HTML and CSS, but with UI frameworks, it looks like it took me just a couple of weeks.
